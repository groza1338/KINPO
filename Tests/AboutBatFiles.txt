# О бат-файлах

Тесты запускаются с помощью start.bat.

Необходимо прописать путь к программе [Gvozdkov_run.exe](..%2FExecutable%2FGvozdkov_run.exe) в первой строке start.bat.

Появится файл results.txt, в котором наглядно отражена информация о каждом тесте.

Перед каждым запуском желательно удалять подкаталог Output и results.txt.

Сначала автоматически запускаются 8 файлов с ошибками во входных данных из подкаталога Errors, во время которых не должны создаваться выходные файлы, а сообщения из консоли записываются в results.txt.

Затем запускаются 9 файлов в подкатологе Input с различными высотами стен, выходные файлы формируются в подкаталоге Output, а их содержимое записывается в results.txt.


## Подкаталог Errors (входные файлы на разные типы ошибок)

error01.txt - Входного файла не существует

[error02.txt](Errors%2Ferror02.txt) - Пустой входной файл.

[error03.txt](Errors%2Ferror03.txt) - Невозможно создать выходной файл.

[error04.txt](Errors%2Ferror04.txt) - Несколько строк во входном файле.

[error05.txt](Errors%2Ferror05.txt) - Входной параметр не является числом.

[error06.txt](Errors%2Ferror06.txt) - Число не в диапазоне [0..4294967295] (отрицательное).

[error07.txt](Errors%2Ferror07.txt) - Число не в диапазоне [0..4294967295] (положительное).

[error08.amogus](Errors%2Ferror08.amogus) - У входного или выходного файла не .txt расширение.



## Подкаталог Input

##### Разные случаи

[input01.txt](Input%2Finput01.txt) - Три стены одинаковой высоты.

[input02.txt](Input%2Finput02.txt) - Стены идут лесенкой.

[input03.txt](Input%2Finput03.txt) - Между стенами есть одна яма.

[input04.txt](Input%2Finput04.txt) - Между стенами две ямы.

[input05.txt](Input%2Finput05.txt) - Две стены.

[input06.txt](Input%2Finput06.txt) - Одна стена.

[input07.txt](Input%2Finput07.txt) - Обратная пирамида из стен.

[input08.txt](Input%2Finput08.txt) - Пирамида из стен.

[input09.txt](Input%2Finput09.txt) - Стены со случайными высотами.
